========== ABSTRACTION & INTERFACE ==========

Abstraction: Hide implementation details; show only essential features.

Abstract class:
- Can have both abstract (no body) and concrete (with body) methods.
- Cannot be instantiated.
- A class can inherit only one abstract class.

Interface:
- No implementation (only signatures of methods, properties).
- Can be implemented by any class (unrelated classes).
- A class can implement many interfaces.

When to use:
- Abstract class: when you have a "is-a" relationship and want to share code.
- Interface: when you need a contract that multiple classes must follow (e.g., sorting, disposal).

Real-world examples in .NET:
- IDisposable → implement to release resources.
- IEnumerable → implement to allow foreach.
- IComparable → implement to allow sorting.
